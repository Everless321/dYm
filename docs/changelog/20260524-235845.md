# 修复 live 图配对在图片/视频数量不一致时整体失效

## 背景

Issue #6 反馈：图集中只有部分图片有对应的 live mp4（例如 image1/image2/image3，仅 live1/live3 存在）时，live 播放功能整体失效，所有图都不会自动播放视频。

## 根因

`src/main/services/media.ts` 中 `findMediaFiles` 对 awemeType=68 的图集做图片↔live 配对时只有两个分支：

1. basename 精确匹配
2. 兜底：basename 全不匹配 **且** 图片数 === 视频数时按顺序配对

当 basename 不匹配（dy-downloader 给图片和 live 的命名风格不同）且数量也不相等时，两个分支同时落空，整个 `imageVideos` 数组全为 `null`，前端无法选择 video 元素渲染。

## 待办

- [x] 重构 `media.ts` 的配对逻辑，引入三级回退
- [x] 保持已有 basename 全匹配场景的行为不变
- [x] 写 changelog

## 修改

`src/main/services/media.ts`：将原 basename + 全量顺序兜底，改为三级回退：

1. **basename 精确匹配**（最强约束）
2. **尾部数字索引匹配**：从文件名提取最后一段数字作为索引（兼容 `image_1` / `1` / `xxx-1` 等多种命名）
3. **位置兜底**：仍未配对的图片，按剩余 mp4 顺序补齐

通过 `usedVideos` Set 防止同一个 mp4 被重复绑定到多张图片。

## 结果

- Issue #6 场景（image1/2/3 + live1/3）：image1→live1, image2→null（回退静图）, image3→live3，符合预期
- 原全匹配场景行为不变
- 仅修改单个函数内 ~15 行，无破坏性变更
