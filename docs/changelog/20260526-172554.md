# 网页端观看体验改进 — 播放器内无限加载 + 退出回到刚看的位置

## 背景

Issue #6 反馈：网页端 18 个一批分页，看完得退到网格里往下滑才能加载更多；退出播放器后又回到「点进去时」的那个视频，丢失了观看进度。

## 根因

`resources/web/app.js`：
- 播放器 `openPlayer()` 只渲染当前 `state.posts`，**没有任何在播放器内部触发加载更多的钩子**
- 唯一的 `loadGrid(false)` 触发点是网格自身的 scroll 监听，进了播放器就摸不到
- `closePlayer()` 只切换显示，不动 grid 滚动位置；`state.activePostId` 在播放器内一直跟随更新但退出时被丢弃

## 待办

- [x] 播放器接近末尾时自动加载下一批，并把新 story 追加进播放器 DOM
- [x] 退出时把网格滚动到「刚看完」的视频并短暂高亮
- [x] 写 changelog + 发版 v1.8.5

## 修改

**`resources/web/app.js`**：
- `loadGrid()` 用 `addedPosts` 收集真正新增的帖子（之前 `incoming` 会重复渲染已存在项），并在播放器开启时调用 `extendPlayer(addedPosts)` 同步追加
- 新增 `extendPlayer(newPosts)`：更新所有 story 的 total 计数器，按新顺序 append story 节点，重新 `bindStories()`
- 新增 `maybeLoadMoreForPlayer(activeId)`：当前激活 story 距末尾 ≤ 3 时触发 `loadGrid(false)`，在 `activateStory()` 末尾调用
- 改写 `closePlayer()`：先记 `lastId = state.activePostId`，切换视图后用 `el.grid.querySelector('[data-post-id]')` 找到对应 grid-item，滚到屏幕中央并加 `is-just-viewed` 高亮 class（1.2s 后移除）

**`resources/web/app.css`**：
- 新增 `.grid-item.is-just-viewed` 高亮样式 + `justViewedPulse` 动画

## 结果

- 播放器内连续上滑：第 16/17 个时自动加载第 19+ 个，无缝衔接
- 退出后网格自动定位并轻微脉冲提示
- 仅改 2 个 web 资源文件，无主进程 / IPC / DB 改动
